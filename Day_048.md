# Day 48: Deploy Pods in Kubernetes Cluster
The Nautilus DevOps team is diving into Kubernetes for application management. One team member has a task to create a pod according to the details below:

* Create a pod named `pod-nginx` using the `nginx` image with the `latest` tag. Ensure to specify the tag as `nginx:latest`.

* Set the `app` label to `nginx_app`, and name the container as `nginx-container`.

> Note: The kubectl utility on jump_host is configured to operate with the Kubernetes cluster.


## Solution
On jump host create a pod.yaml file
```sh
kubectl run pod-nginx --image=nginx:latest --dry_run=client -o yaml > pod.yaml
```

Modify the pod.yaml file like below
```sh
apiVersion: v1
kind: Pod
metadata:
  name: pod-nginx
  labels:
    app: nginx_app
spec:
  containers:
    - name: nginx-container
      image: nginx:latest
      ports:
        - containerPort: 80
      resources:
        limits:
          cpu: 200m
          memory: 256Mi
        requests:
          cpu: 100m
          memory: 128Mi
```


Create the pod resource.
```sh
kubectl create -f pod.yaml
```


Finally verify the pod creation
```sh
kubectl get pods
```
